2
Clone rootAVD về Mac
Lấy đường dẫn ramdisk của máy ảo
TERMINAL
git clone https://github.com/newbit1/rootAVD.git
cd rootAVD

# Liệt kê AVD để lấy đường dẫn ramdisk
sh rootAVD.sh ListAllAVDs

# Kết quả ví dụ:
./rootAVD.sh system-images/android-33/google_apis_playstore/arm64-v8a/ramdisk.img
⚠ Máy ảo Google Play dùng thư mục google_apis_playstore — khác với google_apis. Chú ý lấy đúng đường dẫn từ lệnh ListAllAVDs.
3
Tạo máy ảo Google Play
Android Studio → Virtual Device Manager
1
Mở Virtual Device Manager
Android Studio → biểu tượng điện thoại trên toolbar → Create Device
2
Chọn hardware: Pixel 6
Nhấn Next để tiếp tục
3
Chọn System Image
Tab Recommended → API Level Android API 33 → Target phải có icon ▶ Google Play
4
Đặt tên và khởi động
Đặt tên AVD: Pixel6_API33_GooglePlay → Finish → Start
✓ Máy ảo boot xong, vào CH Play đăng nhập tài khoản Google là thành công. Lúc này chưa root — sẽ thực hiện ở bước tiếp theo.
4
Root máy ảo bằng rootAVD
Magisk — máy ảo phải đang chạy trước
⚠ Máy ảo phải đang chạy trước khi thực hiện. Kiểm tra: adb devices — phải thấy emulator-xxxx device
4.1 — Chạy rootAVD để patch ramdisk
TERMINAL — thay đường dẫn bằng kết quả từ bước 2
./rootAVD.sh system-images/android-33/google_apis_playstore/arm64-v8a/ramdisk.img
  • Tải Magisk APK mới nhất tự động
  • Patch file ramdisk.img của máy ảo
  • Cài Magisk APK lên máy ảo
  • Khởi động lại máy ảo tự động
4.2 — Hoàn tất cài đặt Magisk
a
Mở app Magisk
Sau khi restart, mở Magisk (nếu không mở được → chọn Cold Boot)
b
Additional Setup
Magisk hiện thông báo yêu cầu additional setup → nhấn OK → máy ảo tự reboot thêm 1 lần
c
Xác nhận root
Mở lại Magisk → tab Home → Installed → Direct Install → Let's Go
✓ Magisk hiển thị version ở mục Installed là root thành công.

6.2 — Kết nối máy ảo qua ADB
TERMINAL
# Kiểm tra máy ảo đang chạy
adb devices
emulator-5554 device
  • Trong HTTP Toolkit → Intercept → click Android device via ADB
  • HTTP Toolkit tự cài app + inject CA certificate vào system level
  • Bật VPN để redirect toàn bộ traffic
✓ Vì máy ảo đã root, cả 2 dòng đều tick xanh: USER TRUST ENABLED và SYSTEM TRUST ENABLED. Toàn bộ HTTPS traffic của mọi app sẽ được decrypt.
7
Xem traffic
HTTP Toolkit → tab View
  • Mở app bất kỳ trên máy ảo và dùng bình thường
  • Trên Mac, chuyển sang tab View trong HTTP Toolkit
  • Các request hiện ra theo thời gian thực
HTTP Methods có thể thấy
GET POST PUT DELETE PATCH
Thông tin có thể xem
URL đầy đủ · Headers · Request body (JSON) · Response body · Status code · Thời gian phản hồi
Mẹo filter nhanh
Hành động Tác dụng
Gõ vào ô Search Lọc theo domain hoặc path
Click cột Status Lọc request lỗi 4xx, 5xx
Click từng request Xem đầy đủ header + body
Click Pin source Chỉ xem traffic từ 1 app cụ thể
